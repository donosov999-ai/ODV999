## 1. Текущий статус и решения

- Фреймворк категорий: 10-категорийная схема (последняя итерация), не «колесо жизни 12 секторов».
- Телефонные звонки больше не отдельная категория — маршрутизируются по теме разговора.
- Алиасы людей → канонические имена с явным запретом воспроизводить уничижительные клички (Лидия = «убогая рожа», «енот» и т. п. — только в дословной транскрипции).
- Правила формата (R1–R5) вынесены в начало промпта, до структурных инструкций.
- Единственный разрешённый эмодзи — `📌` в строке «Прочее» по правилу R2.
- Адрес ЖК Амундсен кв. 547: застройщик ГК «Прогресс» (НЕ «Академический») — анти-галлюцинация.

```
═══════════════════════════════════════════════════════════════
PART 2. FORMAT RULES (mandatory, apply BEFORE structure)
═══════════════════════════════════════════════════════════════

R1. Один факт = один блок. Не дублировать содержание между блоками. Один и тот же факт упоминается только в своём «домашнем» блоке; в других блоках на него ссылаются через ID [E0X], без повторного пересказа.

R2. Мимолётное упоминание (одна фраза без деталей, без участников, без последствий) НЕ выносится в отдельный пункт. Все такие мимолётные факты собираются в одну строку в самом конце TOPIC SUMMARIES в формате:
  «📌 Прочее: <короткие упоминания через точку с запятой, без раскрытия> ([E0X], [E0Y]...)».
Эмодзи 📌 — единственный разрешённый эмодзи во всём документе. Используется только как структурный маркер этой строки.

R3. Если ВСЯ запись монотемна (один длинный разговор, одна лекция, один звонок без переключений темы) — структуру не выдумывать. Вывести единым блоком без подзаголовков категорий: META → одна цельная сводка → TO-DO LIST (если есть) → KEY DECISIONS (если есть) → IMPORTANT FACTS (если есть) → OPEN QUESTIONS (если есть) → FULL TRANSCRIPT. Блоки DAY INDEX и группировка TOPIC SUMMARIES по категориям при этом ПРОПУСКАЮТСЯ.

R4. Цитаты — дословно в кавычках «ёлочках», с лейблом спикера и тайм-кодом, если он есть, в формате: Денис: «текст цитаты» (01:23:45).

R5. Если блок пуст — целиком пропустить его, не писать «Нет данных» или «—».

═══════════════════════════════════════════════════════════════
PART 3. MAIN INSTRUCTION
═══════════════════════════════════════════════════════════════

* Source is a continuous voice-recorder file up to 10–15 hours long, with many unrelated episodes: family, business (see PART 1), phone calls on any topic, health, finances, ambient strangers, owner's monologues, silence and noise. Segment the day into episodes first, then provide the full verbatim transcript at the end.

* Write the entire output in Russian. Speakers' words, names and terms stay in the original language without translation.

* Output block order (after applying R3 if monotopic):
  1) META
  2) DAY INDEX
  3) TOPIC SUMMARIES
  4) TO-DO LIST
  5) KEY DECISIONS
  6) IMPORTANT FACTS & CONTACTS
  7) OPEN QUESTIONS
  8) FULL TRANSCRIPT
  No intro, no outro, no meta-commentary. Start with META.

* META: дата записи (если слышна), общая длительность, оценочное число эпизодов, список идентифицированных спикеров с устойчивыми лейблами (используя канонические имена из PART 1).

* DAY INDEX: хронологическая таблица всех эпизодов. Колонки: № | Тайм-код | Длительность | Категория | Участники | Тема одной строкой. Каждому эпизоду — постоянный ID [E01], [E02], ... ссылающийся в остальных блоках.

* Categories (use ONLY these): Работа, Бизнес, Семья, Здоровье, Финансы, Личное / мысли вслух, Бытовое, Окружение / посторонние, Прочее.
  - Телефонные звонки НЕ являются отдельной категорией: маршрутизировать по теме разговора (звонок врачу → Здоровье; Родиону → Бизнес; Вале о саде Алекса → Семья). В колонке "Участники" добавлять "(звонок)".
  - Смешанные эпизоды: "Primary / Secondary" (например, "Бизнес / Здоровье"). Использовать редко.
  - Эпизоды < 30 сек без содержательной речи → объединить в одну строку "Фоновое / нерелевантное" с диапазоном тайм-кодов.

* TOPIC SUMMARIES: сводки сгруппированы ПО КАТЕГОРИЯМ (не хронологически). Подзаголовки "### Категория" только для непустых категорий. Внутри — один абзац на эпизод:
  - **[E0X] Краткое название темы** (тайм-код, участники) — 3–6 предложений сути: что обсуждалось, позиции, итог, важные детали (цифры, имена, даты, договорённости).
  Сохранять имена, числа, суммы, даты, лекарства, адреса, контакты, термины точно. Цитаты — по R4. Разные эпизоды одной темы НЕ объединять. В конце блока — строка "📌 Прочее: ..." по R2, если есть мимолётные упоминания.

* TO-DO LIST: одна таблица на весь день. Колонки: # | Задача | Кто делает | Срок | Категория | Приоритет | Источник | Комментарий.
  - "Кто делает" — "Я" (Денис), конкретное имя или "—".
  - "Срок" — дословно как сказано ("завтра", "до пятницы", конкретная дата) или "—".
  - "Категория" — одна из списка выше.
  - "Приоритет" — Высокий / Средний / Низкий (Высокий = жёсткий дедлайн, деньги, здоровье, явная срочность).
  - "Источник" — [E0X] (несколько через запятую для дублей).
  - Дубли (одна задача в нескольких эпизодах) — одна строка, перечислить источники.
  - Размытые задачи всё равно вносить с "[требует уточнения]" в "Комментарий".

* KEY DECISIONS: нумерованный список решений и договорённостей. Одна строка + [E0X] (без пересказа подробностей — по R1).

* IMPORTANT FACTS & CONTACTS: телефоны, адреса, имена с ролями, лекарства и дозировки, суммы, коды/пароли, компании, ссылки. Формат: "- Факт — короткий контекст — [E0X]". Неуверенные цифры — "[на слух]". Новые лекарства/добавки, не указанные в PART 1, — обязательно сюда. Каждое упоминание Лолиты — обязательно сюда до прояснения роли.

* OPEN QUESTIONS: неотвеченное, оборванное, требующее уточнения. Одна строка + [E0X]. Сюда же — отметки о подозрительных однофамильцах и о «Маркиз [человек]».

* FULL TRANSCRIPT: дословный текст всего сказанного, хронологически. Каждый эпизод открывается подзаголовком "## [E0X] Тайм-код — Категория — Тема". Внутри — реплики "**Спикер:** реплика." с пустой строкой между ходами. Неизвестные — "**Спикер ?:**". Сохранять слова-паразиты, повторы, оборванные фразы. Использовать [неразборчиво], [пауза], [смена обстановки]. Эпизоды "Фоновое / нерелевантное" не включать (только индексная строка). Для "Окружение / посторонние" — только курсивная пометка "_Разговор посторонних людей, услышан случайно._", без слов посторонних.

═══════════════════════════════════════════════════════════════
PART 4. PROHIBITIONS
═══════════════════════════════════════════════════════════════

* Не выдумывать имена, числа, даты, контакты, диагнозы, цитаты, которых нет в записи.
* Не объединять разные эпизоды ради краткости.
* Не пропускать задачи, потому что они «мелкие».
* Не переводить речь на другой язык.
* Не воспроизводить уничижительные клички людей в саммари (см. правило по Лидии).
* Никаких эмодзи, кроме 📌 в строке «Прочее» по R2.
* Никаких вступительных и заключительных фраз от себя.

Formatting: Markdown only. Заголовки ## / ###. Таблицы — для индекса и to-do. Жирный — для ID эпизодов и лейблов спикеров. Курсив — для технических пометок. Тайм-коды — hh:mm:ss.
```

## 6. Игры NZT-48 (8 механик)

1. **Цепочка превращений (Precursor Chain Sprint)** — соло, 60–120 сек. Карта-цель → собрать цепочку с начала, назвать промежуточные продукты + ферменты + кофакторы. Скоринг: +1 за звено, −1 за выдуманное.
2. **Стероидное домино (Cortisol Cascade)** — соло/дуэль. Карты-молекулы стероидогенеза, разложить от холестерина до кортизола/альдостерона/тестостерона. Усложнение: класть карточки ферментов под стрелки.
3. **Кофактор-домино (Cofactor Match)** — соло, 90 сек. Сопоставить ферменты и кофакторы (BH4, PLP, SAMe, витамин C, медь, железо, цинк, магний, NAD+, фолат/B12). Цепочка дефицита: «B6 ↓ → AADC, GAD, кинуренин ломаются».
4. **Бустеры и душители BDNF (BDNF Yes/No)** — соло, 30 сек на карту. Факторы → ↑/↓/нейтрально. Ловушки: «острый стресс» ≠ «хронический», «кофеин» — нейтрально.
5. **Анти-хайпер дуэль: BDNF vs Кортизол (Carded Antagonists)** — пара. По 10 карт у каждого, ходишь — описываешь механизм одним предложением, иначе ход не засчитан. Кто первый довёл BDNF оппонента до 0 или кортизол до 100 — победил.
6. **Циркадный пасьянс (Diurnal Clock)** — соло, медитативно. На циферблате 24 ч разложить пики/провалы: кортизол, мелатонин, дофамин, тестостерон, инсулин, GH, температура, парасимпатика. Усложнение: «кофе в 16:00» сдвигает пик — показать.
7. **Я — молекула (20 вопросов биохак-edition)** — пара. Загадана молекула (дофамин/кортизол/BDNF/серотонин/мелатонин/ГАМК/глутамат/ацетилхолин/NO/BHB/иризин/лептин/грелин/инсулин/IGF-1). Угадывать только через структурные/функциональные признаки.
8. **Хакни цель (Goal Hack Card-Build)** — соло/пара, 3–5 мин. Карта-цель → выбрать 5 интервенций из руки + защитить «механизмом действия». Цели: «поднять BDNF за 7 дней без добавок», «срезать утренний кортизол», «поднять D2-чувствительность», «восстановить ГГН», «либидо без T-замены».

## 8. iPhone Sound Recognition — план интеграции

Не встроенной статистики у iOS нет, но события Sound Recognition можно перехватить через Команды (Shortcuts) → Личная автоматизация → триггер «Распознавание звуков» (iOS 15.4+).

**Стек:**
1. Включить в Настройках → Универсальный доступ → Распознавание звуков нужные категории (минимум: Кошка, Дверной звонок, Стук в дверь, Бытовые приборы, Сигнал автомобиля).
2. Создать в Команды один Shortcut «Log sound event» — принимает параметр (название звука), шлёт POST в Supabase / Make.com / n8n.
3. На каждый включённый звук — отдельная Личная автоматизация, вызывает этот Shortcut с параметром.
4. Перед прогоном Plaud-разбора — SQL-выгрузка событий за день, склеить в текст, подставить в промпт как блок `SOUND-EVENT TIMELINE` ПЕРЕД DAY INDEX.

**Таблица Supabase:**
```sql
create table sound_events (
  id bigserial primary key,
  ts timestamptz not null,
  sound text not null,
  device text,
  lat double precision,
  lon double precision,
  inserted_at timestamptz default now()
);
create index on sound_events (ts);
```

**Блок в промпт (предзаготовка):**
```
SOUND-EVENT TIMELINE (external data, not from the audio)

* The following list is independent sound-detection events from the owner's iPhone for the same day. Use them as auxiliary signals to:
  - confirm episode boundaries (doorbell, alarm, car horn typically mark scene changes);
  - confirm home context (cat sounds = Маркиз = домашняя среда);
  - cross-check timecodes if the recording's timeline looks drifted;
  - enrich META with a one-line summary of detected sounds for the day.
* Never treat sound events as speech and never include them in FULL TRANSCRIPT.
* Event format: HH:MM:SS — sound type — [optional location].
* List:
  <вставляется автоматически перед прогоном>
```

**Известные ограничения:**
- Sound Recognition отключается, когда другое приложение пишет с микрофона. Если Plaud-устройство — отдельная железка, конфликта нет. Если приложение на айфоне — события будут пустые во время записи.
- Ложные срабатывания (тостер → «сирена», блендер → «дрель») — держать сырые события + колонку `verified`.
- Батарея: ~5–10% доп. разряда в сутки на iPhone 15+.

## 9. Что обсуждалось дополнительно

- Критический разбор шаблона (12 пунктов): чанкование Plaud → нестабильность `[E0X]` между чанками, дрейф категорий, лимит длины ответа, ложная диарзация спикеров, etc. Реалистичный совет: «полная транскрипция 15 часов в одном выводе невозможна» — либо отказ от FULL TRANSCRIPT, либо принять обрезание.
- Альтернативная компактная архитектура: META → TO-DO → KEY FACTS → TIMELINE (по 1–2 строки на эпизод), без сквозных ID, с тайм-кодами `@hh:mm:ss` как единственным якорем.
- Приватность: био уйдёт на серверы Plaud с каждым запуском — взвесить чувствительность (точные адреса, диагнозы, стек препаратов, политические амбиции).
